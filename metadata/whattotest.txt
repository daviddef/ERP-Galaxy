Build 21 — mostly navigation, plus a fix for styling that was broken in build 20.

FIXED — MISSING STYLES IN BUILD 20
The field guide, the lessons overlay and the table list all shipped in build 20 with their CSS accidentally deleted, so they rendered as unstyled text. If you looked at any of those in build 20 and thought they looked broken, they were. Worth a second look now.

REBUILT MENU
Three groups instead of eleven stacked blocks, and it fits on one screen:
• TOOLS — a 2x2 grid: To-do (with a count), Find a field, BAPIs, Table list
• LEARN — field guide and lessons as rows; playbooks and legend now fold away
• DISPLAY — Brief / Detailed

MOVED TO THE FILTER ICON
Paste a table list, and Saved views. Both decide what the chart shows, so they sit with the other filters rather than in the menu.

NEW — HALF-HEIGHT PANEL
Open any table, then tap "⇕ Half" in the panel header. The panel parks mid-screen and the dimming comes off, so the graph stays live underneath — tap table after table and the panel just re-fills instead of closing. Tap "⇕ Full" to go back. The choice is remembered.

BAPI SEARCH NOW TAKES BUSINESS LANGUAGE
This is the one I would most like tested. Previously you had to know the exact name. Now try:
• "work order" → BAPI_ALM_ORDER_MAINTAIN and the rest of the PM work-order family
• "goods movement", "purchase order", "inspection lot", "business partner", "cost centre"
Every result shows its SAP application component, so you can judge whether the match is sensible. Please tell me if any of these return something you would not expect — the abbreviation glossary behind it is hand-written and is the part most likely to be wrong.

FLOATING BUTTON
Used to reopen the table detail panel, which tapping a table already does. It now opens your to-do list and shows the count.

TABLE LIST
Its buttons moved above the table — with 400 rows they were pushed off the bottom of the sheet and could not be reached.

STILL WORTH CHECKING FROM BUILD 20 — never run on hardware
• To-do → Export PDF
• Share list, and Share on the field guide
Both compile and the code paths are verified, but neither has ever run on a real device.

AND AS ALWAYS
If a verdict or a description reads wrong to you as a practitioner, that is the most valuable feedback. Several real data errors in these builds were found exactly that way.
