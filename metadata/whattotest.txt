Build 28 — fixes the launch crash in build 27.

FIXED — BUILD 27 CRASHED ON LAUNCH
Apologies. Build 27 crashed for everyone on the first launch after updating.
CoreSpotlight rejects the batching API on the shared index and throws, and the
indexer runs whenever the build number changes — so every tester hit it. The
batching was an unnecessary embellishment and has been removed entirely.
Reproduced and verified fixed in the simulator before cutting this build.

PLEASE CONFIRM FIRST
Just that it launches and stays up. That is the whole point of this build.

THEN, EVERYTHING FROM 27 THAT NOBODY COULD REACH
• Mac and iPad now use the same menus as the phone. Previously any window
  wider than a portrait phone got an older layout: no menu buttons, filters
  and guides spilled permanently inline, a legend stuck over the chart and a
  thin right-hand detail strip.
• iPhone landscape was completely broken — the graph collapsed to nothing.
• 202 tables in the wider index never showed their S/4 verdict. Filter to
  Disappears and tap a table not on the chart — try ANEA, ATAB or COVP.
• Counts that contradicted each other. MARA's joins now read "40 of 106".
• LEFT OUTER JOIN in the ABAP generator. Tap a join, then the code icon.
  The left outer version warns that unmatched columns come back INITIAL
  rather than null, and that a WHERE on the right-hand table silently turns
  it back into an inner join.
• Spotlight search — type a table name into iOS search. Indexed entries were
  expiring after about a month and never rebuilding.
